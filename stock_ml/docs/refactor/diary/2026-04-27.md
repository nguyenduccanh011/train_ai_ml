## Decision

- **Golden baseline dùng CPU mode**, không phải GPU. Lý do: LightGBM GPU (OpenCL) non-det là blocker cho regression test "hash exact" mà Phase 0.2 yêu cầu. Trade-off: training chậm hơn ~20-40% nhưng chấp nhận được — Phase 0.2 chỉ chạy 1 lần để tạo golden, regression tests sau này cũng nhanh.
- **Pipeline runtime production GPU vẫn OK** cho research (không cần exact reproducible). Chỉ regression test bắt buộc CPU.
- **Không modify `_lock_global_seeds`** thêm. PYTHONHASHSEED đã work cho dict/set ordering nhưng không fix được LightGBM GPU OpenCL non-det.
- **Không retry GPU với LightGBM `force_row_wise=True` hay tương tự**. Không có flag fix triệt để cho OpenCL backend, chỉ workaround partial.

## Bug discovered: exit_model wired but unused

User hỏi về exit_model + tại sao score >600. Audit golden: tất cả 11 champions có
`exit_reason='model_b_exit'` count = 0 → exit_model trained nhưng output không bao giờ vào
backtest.

Root cause: [run_pipeline.py:361](../../run_pipeline.py#L361) gate
`"y_pred_exit" in sig_params` chỉ match wrapper có khai báo y_pred_exit explicit. 9/10 modern
wrapper dùng `(*y_pred, returns, df_test, feature_cols, **kwargs)` → gate fail → drop. Engine
nhận None → bỏ qua code path model_b_exit ở
[engine.py:737-740](../../src/backtest/engine.py#L737-L740).

CHAMPION_VERSIONS.md mô tả v37a_exit, v42_a "exit_model thực sự active" — sai. Cả 2 cùng 0
trade model_b_exit.

Score >600 không bị inflate — kết quả phản ánh đúng entry model + fusion strategies khác
(fast_exit_loss, signal_hard_cap, v32_hap_preempt, peak_protect_ema, …). Không nhầm lẫn data
leak.

Decision: KHÔNG fix ngay. Document ở [EXIT_MODEL_BUG.md](../EXIT_MODEL_BUG.md). Fix triệt để
ở Phase 2 khi rebuild fusion stack với `ml_exit_model` là explicit strategy. Golden hiện tại
giữ nguyên — phản ánh đúng pre-refactor behavior (bug và tất cả).

Lãng phí compute: ~1000s/run training Model B vứt đi. Sau Phase 2 có thể bỏ Model B training
nếu audit cho thấy không có predictive power.
