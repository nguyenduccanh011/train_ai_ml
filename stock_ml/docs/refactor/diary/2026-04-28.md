## Stuck

- **Windows file lock** với ruff format (3-5 lần). Pattern: `os error 1224 user-mapped section
  open`. Workaround: sleep 2-5s rồi retry. Nguyên nhân giả thuyết là antivirus/Windows Search
  hold file briefly. Không phải blocker, chỉ làm chậm 1 phút.
- `pre-commit install` lần đầu fail vì `core.hooksPath` đã set tới default `.git/hooks` →
  pre-commit từ chối (cowardly refusing). Unset → install OK.

## Decision

- **Per-file-ignores rộng cho legacy code thay vì full strict**: 244/424 ruff errors ở
  `src/strategies/legacy.py` toàn `SIM102` (collapsible-if). Strict toàn bộ = 1 ngày auto-fix +
  3 ngày verify behavior + regen golden. Trade-off: legacy giữ nguyên (sẽ migrate qua components
  Phase 1+), code mới strict full ruleset. Đúng spirit roadmap "không backwards compat
  burden".
- **Regression-champions hook chỉ hash check, không rerun pipeline**: rerun mỗi commit (~6m) =
  block dev flow. Hash check ~1s, pre-commit nhanh. User regen pipeline thủ công sau khi sửa
  src/. Trade-off: nếu code thay đổi behavior nhưng không regen → hook không catch. Acceptable
  vì refactor target là "không thay đổi behavior", regen là phần ý thức của workflow.
- **Branch không tách phase-0.3**: tooling setup overhead < 1 ngày, sub-branch không value-add.
  Tiếp tục refactor/phase-0.2-golden cho Phase 0.3-0.5.
- **Mypy chưa enable strict cho src/ legacy**: `files = ["stock_ml/src/components", "stock_ml/
  src/pipeline"]` — chưa exist ở Phase 0. Nghĩa là mypy chạy 0 errors thoải mái cho tới Phase 1.

## Risk note for downstream phases

- Khi Phase 1.2 viết feature blocks vào `src/components/features/`, mypy sẽ active strict —
  type hint phải đầy đủ từ đầu, không lazy.
- Khi Phase 2.x port fusion strategies, nhớ regen golden + verify hash sau mỗi merge — pre-commit
  hook chỉ catch hash mismatch nếu đã regen. Nếu refactor đổi format CSV (vd: thêm column) →
  regen golden + commit checksums.txt mới.

  name). Nếu sau này cleanup `experiments/` (CLEANUP_PLAN.md step 6), có thể siết lại.
